Ringkasan Materi: Selenium

1. Apa Itu Selenium?

Selenium adalah framework open-source yang portabel yang digunakan untuk otomatisasi pengujian aplikasi web.

Secara sederhana, Selenium bukanlah satu alat saja, melainkan sebuah rangkaian (suite) perangkat lunak yang memungkinkan Anda menulis skrip untuk mengontrol tindakan browser secara otomatis. Anda bisa memerintahkan skrip untuk membuka browser, mengeklik tombol, mengisi formulir, dan memverifikasi teks di halaman, seolah-olah dilakukan oleh pengguna sungguhan.

Tujuan utamanya adalah untuk software testing (pengujian perangkat lunak), khususnya automated regression testing (memastikan fitur lama tidak rusak setelah ada perubahan kode baru).

2. Kenapa Menggunakan Selenium? (Keunggulan)

  • Gratis dan Open-Source: Tidak memerlukan biaya lisensi. Siapa pun dapat menggunakannya dan berkontribusi.
  • Dukungan Multi-Bahasa: Anda bisa menulis skrip tes menggunakan Python, Java, C#, JavaScript, atau Ruby.
  • Dukungan Multi-Browser: Skrip dapat dijalankan di berbagai browser seperti Google Chrome, Mozilla Firefox, Safari, dan Microsoft Edge.
  • Dukungan Multi-Platform: Skrip Selenium dapat berjalan di Windows, macOS, dan Linux.
  • Komunitas Besar & Ekosistem Matang: Banyak tutorial, forum, dan library pendukung untuk integrasi (seperti TestNG, PyTest, JUnit).

3. Komponen Utama Selenium

A. Selenium WebDriver )

WebDriver adalah API (Application Programming Interface) yang memungkinkan skrip berkomunikasi langsung dengan browser. Setiap browser memiliki "penerjemah" sendiri yang disebut Browser Driver (misalnya chromedriver.exe untuk Chrome, geckodriver.exe untuk Firefox).

B. Selenium IDE

Ini adalah plugin browser (untuk Chrome dan Firefox) yang bekerja dengan model record-and-playback.

C. Selenium Grid

Ini digunakan untuk skalabilitas pengujian. Selenium Grid memungkinkan menjalankan tes secara paralel (bersamaan) di beberapa mesin, browser, dan sistem operasi yang berbeda sekaligus untuk menghemat waktu.

4. Konsep Kunci dalam Praktik

A. Web Elements & Locators

Web Elements adalah "benda" di halaman (tombol, link, form). Locators (Selectors) adalah "alamat" atau cara Selenium untuk menemukan elemen tersebut. Ini adalah bagian paling krusial.

Tipe Locator Prioritas Deskripsi
ID Tinggi Paling cepat dan paling stabil. Selalu gunakan jika ada dan unik.
Name Sedang Sering digunakan untuk elemen formulir (input, textarea).
Class Name Sedang Berguna untuk menemukan elemen dengan gaya (CSS) yang sama.
CSS Selector Tinggi Sangat fleksibel dan cepat, cara modern untuk menemukan elemen.
XPath Rendah Paling kuat (bisa navigasi struktur HTML), tetapi lambat dan rapuh.
Link Text Rendah Hanya untuk menemukan tautan (elemen <a>) berdasarkan teksnya.

B. Waits

Halaman web modern bersifat dinamis. Elemen mungkin belum muncul saat skrip mencarinya. Waits memberi tahu skrip untuk menunggu.

  • Implicit Wait: Perintah global untuk menunggu sekian detik sebelum gagal jika elemen tidak ditemukan.
  • Explicit Wait (Direkomendasikan): Perintah spesifik untuk menunggu sampai kondisi tertentu terpenuhi (misalnya, WebDriverWait hingga tombol bisa diklik).

5. Keterbatasan Selenium

  • Bukan untuk Aplikasi Desktop: Hanya untuk aplikasi berbasis web di dalam browser.
  • Bukan untuk Aplikasi Mobile: Untuk aplikasi native Android/iOS, gunakan Appium
  • Tidak Bisa Menangani CAPTCHA: Selenium tidak dirancang untuk mem-bypass tes keamanan.
  • Bukan Alat Pelaporan: Perlu diintegrasikan dengan framework pengujian (PyTest, TestNG) untuk membuat laporan tes yang bagus.

Contoh Praktikum: Login Sederhana

Berikut adalah contoh skrip Selenium dasar menggunakan Python. Skenario ini akan melakukan langkah-langkah berikut:

  1. Membuka browser Chrome.
  2. Menavigasi ke halaman login (kita gunakan http://quotes.toscrape.com/login sebagai contoh).
  3. Menemukan elemen *username* dan *password* berdasarkan ID-nya.
  4. Mengisi formulir.
  5. Menemukan tombol "Login" dan mengekliknya.
  6. Memverifikasi bahwa login berhasil dengan memeriksa teks di halaman berikutnya.
  7. Menutup browser.

Pastikan Anda sudah menginstal Selenium (pip install selenium) dan chromedriver yang sesuai ada di PATH Anda atau dikelola oleh webdriver-manager. 

# 1. Impor library yang diperlukan
from selenium import webdriver
from selenium.webdriver.common.by import By
from selenium.webdriver.support.ui import WebDriverWait
from selenium.webdriver.support import expected_conditions as EC
import time

# 2. Inisialisasi WebDriver
# Cara modern (Selenium 4+) tidak perlu setup driver manual
driver = webdriver.Chrome()
wait = WebDriverWait(driver, 10) # Set Explicit Wait (max 10 detik)

# 3. Buka halaman web
driver.get("http://quotes.toscrape.com/login")
print("Halaman login dibuka...")

# 4. Cari elemen, isi form, dan klik login
try:
    # Tunggu sampai elemen 'username' bisa diakses
    username_field = wait.until(EC.presence_of_element_located((By.ID, "username")))
    password_field = driver.find_element(By.ID, "password")
    
    username_field.send_keys("admin")
    password_field.send_keys("admin")
    print("Formulir diisi...")

    # Klik tombol login (menggunakan CSS Selector)
    login_button = driver.find_element(By.CSS_SELECTOR, "input[type='submit']")
    login_button.click()
    print("Tombol login diklik.")

    # 5. Verifikasi hasil
    # Tunggu hingga URL berubah atau elemen logout muncul
    wait.until(EC.presence_of_element_located((By.LINK_TEXT, "Logout")))
    
    # Ambil teks dari elemen 'Logout' untuk konfirmasi
    logout_link = driver.find_element(By.LINK_TEXT, "Logout")
    
    if logout_link.is_displayed():
        print("Verifikasi BERHASIL: Login sukses.")
    else:
        print("Verifikasi GAGAL: Login tidak berhasil.")

except Exception as e:
    print(f"Terjadi error: {e}")

finally:
    # 6. Tutup browser
    time.sleep(3) # Beri jeda 3 detik untuk melihat hasil
    driver.quit()
    print("Browser ditutup.")
